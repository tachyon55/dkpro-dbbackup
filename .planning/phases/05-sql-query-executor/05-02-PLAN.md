From src/lib/db-drivers/index.ts (created in 05-01):
```typescript
export interface QueryResult {
  type: "select" | "dml" | "other"
  columns?: string[]
  rows?: Record<string, unknown>[]
  rowCount?: number
  durationMs: number
  capped?: boolean
}

export const ROW_CAP = 500

export function detectStatementType(sql: string): "select" | "dml" | "other"
```

API: POST /api/query/execute
- Request: `{ connectionId: string, sql: string }`
- Success: `{ data: QueryResult }`
- Error 400: `{ error: string }` (DB error message)
- Error 401: `{ error: "인증이 필요합니다" }`
- Error 403: `{ error: "viewer 역할은 SELECT만 실행할 수 있습니다" }`
- Error 404: `{ error: "연결을 찾을 수 없습니다" }`

API: GET /api/query/saved
- Response: `{ data: SavedQuery[] }`

API: POST /api/query/saved
- Request: `{ name: string, sql: string, connectionId?: string }`
- Response 201: `{ data: SavedQuery }`

API: PUT /api/query/saved/[id]
- Request: `{ name?: string, sql?: string, connectionId?: string | null }`
- Response: `{ data: SavedQuery }`

API: DELETE /api/query/saved/[id]
- Response: `{ data: { id: string } }`

SavedQuery shape from Prisma:
```typescript
{ id: string, userId: string, connectionId: string | null, name: string, sql: string, createdAt: string, updatedAt: string }
```

Connection shape passed from server component:
```typescript
{ id: string, name: string, type: DbType, color: string | null }
```

**`src/app/(app)/query/page.tsx`** — Server Component shell:
- `import { auth } from "@/lib/auth"`, `import { redirect } from "next/navigation"`, `import { prisma } from "@/lib/prisma"`
- Auth guard: redirect to /login if no session
- Fetch connections: `prisma.dbConnection.findMany({ orderBy: { name: "asc" }, select: { id: true, name: true, type: true, color: true } })`
- Render `<QueryPageClient connections={connections} userRole={session.user.role} />`
- Pattern: identical to existing connections/page.tsx server component pattern

**`src/components/query/QueryEditor.tsx`** — Monaco wrapper:
- `"use client"` with dynamic import: `import dynamic from "next/dynamic"` and `const Editor = dynamic(() => import("@monaco-editor/react").then(m => m.default), { ssr: false })` — Monaco cannot render server-side
- Props: `{ value: string, onChange: (sql: string) => void, onExecute: () => void }`
- Monaco options per UI-SPEC 7.2: `minimap: { enabled: false }, fontSize: 14, lineNumbers: "on", scrollBeyondLastLine: false, wordWrap: "on"`
- `height="200px"`, `defaultLanguage="sql"`, `theme="vs"`
- Ctrl+Enter binding: use `onMount` callback to register `editor.addCommand(monaco.KeyMod.CtrlCmd | monaco.KeyCode.Enter, onExecute)` per UI-SPEC 11
- Wrapper div: `rounded-md border border-neutral-200 overflow-hidden`

**`src/components/query/ResultTable.tsx`** — Result display:
- Props: `{ result: QueryResult | null, error: string | null, isLoading: boolean }`
- 4 states per UI-SPEC 7.5:
  1. **Empty** (result===null and !error and !isLoading): Terminal icon + "쿼리를 실행하면 결과가 여기에 표시됩니다" centered in py-12
  2. **Loading** (isLoading): 5 Skeleton rows
  3. **Error** (error): red-50 block with AlertCircle icon + error message, `role="alert"`
  4. **SELECT** (result.type==="select"): Meta row (Clock icon + durationMs + row count + capped warning in amber-600) + shadcn Table with columns from result.columns, rows from result.rows. NULL values: `<span className="text-neutral-400 italic">NULL</span>`. Cell class: `font-mono text-xs`. Table wrapper: `overflow-auto max-h-96 rounded-md border border-neutral-200`
  5. **DML** (result.type==="dml" or "other"): CheckCircle2 icon + "{rowCount}행이 영향을 받았습니다" + "실행 시간: {durationMs}ms"

**`src/components/query/SavedQueryPanel.tsx`** — Saved query list:
- Props: `{ queries: SavedQuery[], connections: Connection[], onLoad: (sql: string, connectionId: string | null) => void, onDelete: (id: string) => void, isDeleting: string | null }`
- Empty state per UI-SPEC 7.7: Save icon + "저장된 쿼리가 없습니다"
- Table columns: 이름 | 연결 | 저장일 | 액션
- Row click calls `onLoad(query.sql, query.connectionId)` — loads SQL into editor + sets connection
- Connection column: color dot + name, or "범용" in neutral-400 if null
- Date: `format(createdAt, "yyyy-MM-dd")` with date-fns
- Delete button: Trash2 ghost icon button with `aria-label="저장된 쿼리 삭제"`
- Delete uses AlertDialog confirmation per UI-SPEC 7.7

**`src/components/query/QueryPageClient.tsx`** — Main orchestrator:
- `"use client"`
- Props: `{ connections: Connection[], userRole: string }`
- State per UI-SPEC section 8: selectedConnectionId, sql, isLoading, result, error, savedQueries, saveModalOpen, deleteTargetId, activeTab
- `useEffect` on mount: fetch GET /api/query/saved -> setSavedQueries
- `handleExecute()`: validate connection+sql (sonner toast if empty per UI-SPEC 9), POST /api/query/execute, set result or error, switch to results tab
- `handleSave()`: POST /api/query/saved with name + sql + connectionId, close modal, toast "쿼리가 저장되었습니다", refresh list
- `handleDelete(id)`: DELETE /api/query/saved/{id}, toast "쿼리가 삭제되었습니다", remove from list
- `handleLoad(sql, connectionId)`: set sql + selectedConnectionId, switch to results tab
- DDL warning: per UI-SPEC 7.4 — detect non-SELECT client-side, show amber banner for operator/admin only. Use simple startsWith check (same logic as detectStatementType but client-side, UX only)
- Layout per UI-SPEC section 5:
  - `div className="p-6"` container
  - Header row: title "SQL 쿼리" (text-lg font-semibold) + Select connection picker (w-[200px]) with color dots
  - Toolbar: Play icon "쿼리 실행" button (default variant, disabled when no connection/empty sql/loading, shows Loader2 spinner when loading) + Save icon "저장" button (outline, disabled when sql empty)
  - DDL warning banner (conditional)
  - QueryEditor
  - Tabs bottom panel: "결과" tab -> ResultTable, "저장된 쿼리" tab -> SavedQueryPanel
- Save modal: shadcn Dialog with "쿼리 저장" title, name Input, connection Select (pre-filled, clearable with "연결 없음 (범용 쿼리)" option), SQL preview in pre block, 저장/취소 buttons
