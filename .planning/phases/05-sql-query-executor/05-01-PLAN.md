  <action>
**In `src/lib/db-drivers/index.ts`**, add to existing exports (do NOT remove testConnection/listDatabases):

1. `QueryResult` interface: `{ type: "select" | "dml" | "other", columns?: string[], rows?: Record<string, unknown>[], rowCount?: number, durationMs: number, capped?: boolean }`
2. `ROW_CAP = 500` const
3. `detectStatementType(sql: string): "select" | "dml" | "other"` — checks trimmed uppercase first keyword: SELECT/WITH -> "select", INSERT/UPDATE/DELETE/MERGE -> "dml", else "other". Use regex `/^(INSERT|UPDATE|DELETE|MERGE)\s/` for DML.
4. `executeQuery(config: DbConnectionConfig, sql: string): Promise<QueryResult>` — switch on config.type with dynamic imports (same pattern as testConnection). Apply ROW_CAP after driver returns: if rows.length > ROW_CAP, slice and set capped=true.
5. BigInt serializer helper: `serializeRows(rows)` — iterate and convert any BigInt values to strings before returning.

**In each driver file**, add an execute function following RESEARCH.md patterns exactly:

**mysql.ts** — `executeMysql(config, sql): Promise<QueryResult>`:
- `mysql2/promise` createConnection with `connectTimeout: 10000`
- `const [rows] = await conn.execute(sql)` in try/finally with `conn.end()`
- If `Array.isArray(rows)` -> SELECT: columns from `Object.keys(rows[0])`, rows as-is
- Else -> DML: `(rows as ResultSetHeader).affectedRows`
- Pitfall 5: check Array.isArray to distinguish SELECT from DML

**postgres.ts** — `executePostgres(config, sql): Promise<QueryResult>`:
- `new Client()` with `connectionTimeoutMillis: 10000`
- `client.query({ text: sql, query_timeout: 30000 })` in try/finally with `client.end()`
- If `result.fields.length > 0` -> SELECT: columns from `result.fields.map(f => f.name)`
- Else -> DML: `result.rowCount ?? 0`

**mssql.ts** — `executeMssql(config, sql): Promise<QueryResult>`:
- `mssql.connect()` with `requestTimeout: 30000`
- `pool.request().query(sql)` in try/finally with `pool.close()`
- If `result.recordset?.length >= 0` -> SELECT: columns from `Object.keys(result.recordset[0] || {})`
- DML: `result.rowsAffected[0] ?? 0` — Pitfall 4: rowsAffected is array

**sqlite.ts** — `executeSqlite(config, sql): Promise<QueryResult>`:
- Open with `readonly: true` for SELECT, `readonly: false` for DML/other (Pitfall 3)
- Use `detectStatementType` to determine mode
- SELECT: `db.prepare(sql).all()` -> columns from Object.keys of first row
- DML: `db.prepare(sql).run()` -> `result.changes`
- Wrap in try/finally with `db.close()`

**oracle.ts** — `executeOracle(config, sql): Promise<QueryResult>`:
- Dynamic import oracledb, set `outFormat: oracledb.OUT_FORMAT_OBJECT` (Pitfall 1)
- `connection.execute(sql, [], { outFormat })` with try/finally `connection.close()`
- Wrap in `Promise.race()` with 30s timeout (Oracle callTimeout uncertainty — per RESEARCH open question)
- SELECT: columns from `result.metaData.map(m => m.name)`, rows from result.rows
- DML: `result.rowsAffected`

All durations: `Date.now() - start` pattern. All functions export with named export.

  <name>Task 2: Create query execution + saved query CRUD API routes</name>
  <files>src/app/api/query/execute/route.ts, src/app/api/query/saved/route.ts, src/app/api/query/saved/[id]/route.ts</files>
  <action>
**POST /api/query/execute** (`src/app/api/query/execute/route.ts`):
1. Auth check: `const session = await auth()` — 401 if no session
2. Zod validate body: `{ connectionId: z.string(), sql: z.string().min(1).max(10000) }` — 400 on invalid
3. Fetch connection: `prisma.dbConnection.findUnique({ where: { id: body.connectionId } })` — 404 if not found
4. RBAC: `detectStatementType(body.sql)` — if `session.user.role === "viewer"` and type !== "select", return 403 with "viewer 역할은 SELECT만 실행할 수 있습니다"
5. Decrypt password: `connection.password ? decrypt(connection.password) : null`
6. Build `DbConnectionConfig` from connection record (map Prisma fields to config interface)
7. `const result = await executeQuery(config, body.sql)` in try/catch — 400 on error with `err.message`
8. Return `NextResponse.json({ data: result })`
- Import `auth` from `@/lib/auth`, `prisma` from `@/lib/prisma`, `decrypt` from `@/lib/crypto`, `executeQuery`/`detectStatementType` from `@/lib/db-drivers`

**GET /api/query/saved** (`src/app/api/query/saved/route.ts`):
1. Auth check — 401
2. `prisma.savedQuery.findMany({ where: { userId: session.user.id }, orderBy: { updatedAt: "desc" }, include: { /* no include needed — connectionId is enough */ } })`
3. Return `{ data: queries }`

**POST /api/query/saved** (same file):
1. Auth check — 401
2. Zod validate: `{ name: z.string().min(1).max(100), sql: z.string().min(1), connectionId: z.string().optional() }`
3. `prisma.savedQuery.create({ data: { userId: session.user.id, name, sql, connectionId: connectionId || null } })`
4. Return 201 with `{ data: query }`

**PUT /api/query/saved/[id]** (`src/app/api/query/saved/[id]/route.ts`):
1. Auth check — 401
2. `const { id } = await params`
3. Ownership guard: `prisma.savedQuery.findUnique({ where: { id, userId: session.user.id } })` — 404 if not found (Pitfall 6)
4. Zod validate: `{ name: z.string().min(1).max(100).optional(), sql: z.string().min(1).optional(), connectionId: z.string().nullable().optional() }`
5. `prisma.savedQuery.update({ where: { id }, data: { ...validated } })`
6. Return `{ data: updated }`

**DELETE /api/query/saved/[id]** (same file):
1. Auth check — 401
2. Ownership guard (same as PUT) — 404 if not found
3. `prisma.savedQuery.delete({ where: { id } })`
4. Return `{ data: { id } }`

All routes follow the exact auth/error pattern from existing `src/app/api/connections/route.ts`. All error messages in Korean. Use `NextResponse.json()` throughout.
