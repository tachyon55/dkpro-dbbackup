## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge inner padding |
| sm | 8px | Compact row padding, button icon gap |
| md | 16px | Card body padding (matches `p-4` on ConnectionCard) |
| lg | 24px | Section internal padding, modal body padding |
| xl | 32px | Between major sections within a page |
| 2xl | 48px | Page-level vertical rhythm |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Backup button height: 28px (`h-7`) — component height constraint (not a layout spacing token; does not affect grid alignment). Matches existing dropdown trigger sizing on ConnectionCard for Phase 1 continuity.
- Connection card fixed height: 140px — established in Phase 1, do not change
- Touch targets minimum: 44px height for primary action buttons in BackupProgressModal

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 14px (`text-sm`) | 400 (normal) | 1.5 | Geist Sans | Table cells, modal body text, filter labels |
| Label | 12px (`text-xs`) | 600 (semibold) | 1.4 | Geist Sans | Badge text, column headers, metadata (file size, duration) |
| Heading | 16px (`text-base`) | 600 (semibold) | 1.25 | Geist Sans | Modal title, page section headers, detail panel title |
| Mono | 12px (`text-xs`) | 400 (normal) | 1.6 | Geist Mono | Log stream lines in BackupProgressModal, SHA-256 hash display |

**Rule:** Exactly 2 weights across the system: 400 (normal) for Body and Mono roles; 600 (semibold) for Label and Heading roles. Never introduce a third weight (e.g., 500 medium) on any surface.

**New component.** Opens immediately after confirm. Wraps shadcn `Dialog`. Cannot be dismissed while backup is running (`[preventClose]` during in-progress state).

**Layout — top to bottom:**

```
┌─────────────────────────────────────┐
│ [Connection name] 백업 진행 중       │  ← Dialog title, text-base semibold
│ MySQL  •  db_name                   │  ← text-xs neutral-500 badges
├─────────────────────────────────────┤
│ ████████████░░░░░░░░░░░░  IN_PROGRESS│  ← Progress bar (shadcn Progress)
│ 단계: 덤프 실행 중...                │  ← text-xs neutral-500, stage label
├─────────────────────────────────────┤
│ [Log stream area — monospace]       │  ← font-mono text-xs leading-relaxed
│ scrollable, max-h-64, bg-neutral-950│  ← dark terminal aesthetic
│ text-neutral-200                    │
│ stderr lines: text-amber-400        │
│ error lines:  text-red-400          │
└─────────────────────────────────────┤
│ [Result summary — shown on complete]│  ← Appears after stream closes
│ 상태: 성공 / 실패                    │  ← Badge
│ 파일 크기: 12.4 MB                  │  ← text-sm
│ 소요시간: 00:01:23                  │  ← text-sm
└─────────────────────────────────────┤
│                          [닫기]      │  ← Enabled only after complete/failed
└─────────────────────────────────────┘
```

| State | Progress Bar | Log Area | Close Button |
|-------|-------------|----------|--------------|
| STARTED | 10% indigo fill | "연결 중..." log line | Disabled |
| IN_PROGRESS | 50% indigo fill (indeterminate-style) | Streaming stdout/stderr | Disabled |
| COMPLETED | 100% green fill | Final log lines | Enabled, primary |
| FAILED | 100% red fill (`bg-red-500`) | Error lines in red-400 | Enabled, outline |

**Dialog width:** `max-w-2xl`. Log area: `h-64 overflow-y-auto` with auto-scroll to bottom on new lines.

**New component.** Opens on row click. Renders as a right-side panel (`Sheet` from shadcn, side="right", width `w-[480px]`).

**Layout — top to bottom:**

```
├─────────────────────────┤

│ [hash value monospace]  │  ← font-mono text-xs break-all
│                [복사]   │  ← ghost button, `Copy` icon + "복사", text-xs
├─────────────────────────┤
│ [다운로드] button       │  ← full-width, variant="default" indigo
│                         │     `Download` icon + "파일 다운로드"
├─────────────────────────┤
│ 실행 로그               │  ← section label
│ [scrollable log area]   │  ← font-mono text-xs, bg-neutral-950,
```

**Download button states:**

| State | Visual |
|-------|--------|
| File exists | `variant="default"` — "파일 다운로드" |
| File missing (deleted from disk) | `variant="outline"` disabled — "파일을 찾을 수 없음" |
| 진행중 status | Button hidden entirely |

**Failed backup:** SHA-256 section hidden. Download button hidden. Error message displayed in `bg-red-50 border border-red-200 rounded p-3 text-sm text-red-700` box above log section.

## Interaction Contracts

### Backup Execution Flow (D-01 → D-07)

```
ConnectionCard "백업 실행" click
  → BackupConfirmDialog opens
  → User clicks "실행"
  → POST /api/backups (returns jobId)
  → BackupProgressModal opens immediately
  → GET /api/backups/{jobId}/stream (SSE)
  → SSE events drive: progress bar % + stage label + log lines
  → On SSE 'complete': show result summary, enable "닫기"
  → On SSE 'error': show error summary (red), enable "닫기"
  → User clicks "닫기" → modal closes
  → ConnectionCard button re-enables
```

**Cannot close modal during backup** — Dialog `onOpenChange` ignores close events while status is IN_PROGRESS or STARTED.

### History Navigation Flow (D-08 → D-12)

```
Sidebar "백업 히스토리" click
  → /backup-history page loads
  → Table renders with default filters (all connections, all status, last 30 days)
  → User applies filter → URL query params update → table re-fetches
  → Row click → BackupDetailPanel (Sheet) opens from right
  → "파일 다운로드" click → browser initiates download via /api/backup-history/{id}/download
  → "복사" click → SHA-256 to clipboard, button label "복사됨" 1500ms
```

### Concurrency Guard (D-03)

When a backup is in-progress for a connection, the "백업 실행" button on that specific ConnectionCard:
- Becomes disabled (`disabled` attribute)
- Shows `Loader2 animate-spin` icon
- Label: "백업 중..."
- Color: `text-amber-600`
- Other connection cards are unaffected

State is tracked client-side via a React state set of in-progress `connectionId` values, updated by SSE events.
