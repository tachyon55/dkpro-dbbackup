## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps (`gap-1`), inline badge padding (`px-1.5 py-0.5`) |
| sm | 8px | Compact element spacing, card internal gaps (`gap-2`, `p-2`) |
| md | 16px | Default element spacing, card padding (`p-4`) |
| lg | 24px | Section padding, page heading margin-bottom (`mb-6`) |
| xl | 32px | Layout gaps between major page sections |
| 2xl | 48px | Major section breaks (page top padding) |
| 3xl | 64px | Page-level vertical rhythm (not commonly used) |

Exceptions:
- Monaco editor height: fixed `h-64` (256px) — not on the 8-point scale, dictated by usability
- Metric cards: fixed `h-28` (112px) — matches dashboard mockup proportion
- Touch target minimum: 44px for all interactive icon buttons (applies to toolbar buttons in query page)

## Color

| Role | Value | Tailwind | Usage |
|------|-------|----------|-------|
| Dominant (60%) | #ffffff | `bg-white` | Page backgrounds, card surfaces |
| Secondary (30%) | #f5f5f5 / neutral-50–100 | `bg-neutral-50`, `bg-neutral-100` | Sidebar (`bg-white border-r`), table row alternation, badge backgrounds |
| Accent (10%) | indigo-600 (#4f46e5) | `text-indigo-600`, `bg-indigo-50` | Active nav item, primary CTA buttons, links, sidebar active state |
| Success | green-700 / green-100 | `text-green-700 bg-green-100` | Backup success badge, schedule active badge |
| Warning / Destructive | red-600 / red-50 | `text-red-600 bg-red-50` | Failure badge, metric card failure count, delete actions |
| Amber (in-progress) | amber-600 | `text-amber-600` | Backup-in-progress state (established in Phase 2) |

Source: Confirmed from `Sidebar.tsx` (`text-indigo-600`, `bg-indigo-50`), `ConnectionCard.tsx` (green/neutral badges, red delete item).

Accent reserved for:
- Active sidebar navigation item background + text
- Primary action buttons (백업 실행, SQL 실행, 저장)
- Hypertext links (스케줄 설정하기)
- Focus ring on form inputs

Accent is NOT used for:
- Informational badges (use neutral)
- Secondary buttons (use outline variant)
- Table header backgrounds

Dark mode: globals.css defines `--background: #0a0a0a` for `prefers-color-scheme: dark`. Phase 4 components should use CSS variable tokens (`bg-background`, `text-foreground`) for all surfaces so dark mode works without override.

## Page-Level Layout Contracts

### /dashboard

```
┌─────────────────────────────────────────────────────────┐
│ Page heading: "대시보드"  text-xl font-semibold mb-6     │
├──────────┬──────────┬──────────┬──────────────────────────┤
│ 총 연결  │ 오늘 성공 │ 실패 건수 │     다음 스케줄            │
│  Card    │  Card    │  Card*   │      Card                 │
│ h-28     │  h-28    │ h-28 red │      h-28                 │
├─────────────────────────┬───────────────────────────────────┤
│ 연결별 최근 백업 상태     │  최근 백업 히스토리 (10건)          │
│ (grid: name, type,      │  (table: 연결명, 상태, 파일명,      │
│  status, time)          │   크기, 시간)                      │
└─────────────────────────┴───────────────────────────────────┘
```

- 4 metric cards: `grid grid-cols-4 gap-4`
- Bottom panels: `grid grid-cols-2 gap-4 mt-6`
- Failure card highlight: when count > 0, card border-left `4px solid #dc2626` (red-600), number in `text-red-600 text-2xl font-bold`
- Failed rows in connection grid: `bg-red-50` row background
- Data is fetched server-side (Server Component direct Prisma query — Claude's Discretion decision)
- Skeleton loading: render `<Skeleton>` for all 4 cards and both panels during hydration

### /query/[connectionId]

```
┌────────────────────────────────────────────────────────────┐
│ ← 연결 관리   |   SQL 쿼리 실행기   |   [Connection Select ▼] │
├────────────────────────────────────────────────────────────┤
│  Monaco Editor  (h-64, SQL language, Geist Mono 14px)       │
│                                                            │
│  [SQL 실행  ▶]  [저장  💾]  [저장된 쿼리 ▼]  실행시간: 42ms  │
├──────────────────────────────────┬─────────────────────────┤
│ Tabs: [결과] [저장된 쿼리]         │                         │
│  결과: table (rows) or DML msg    │                         │
└────────────────────────────────────────────────────────────┘
```

- Back link: `← 연결 관리` as `text-sm text-neutral-600 hover:text-neutral-900` at page top-left
- Connection select: `<Select>` component, full connection list, pre-selected from URL param `[connectionId]`
- Editor: Monaco Editor, `language="sql"`, `theme="vs"` (light) or `"vs-dark"` matching system color scheme, `height="256px"`, `fontFamily="Geist Mono"`, `fontSize=14`
- Toolbar row: `flex items-center gap-3 mt-3` below editor
- Execute button: primary `<Button>` with `Play` icon — indigo accent, operator+ only
- Save button: `<Button variant="outline">` with `Save` icon
- Saved queries dropdown: `<Select>` populated from user's saved queries
- Execution time: `text-xs text-neutral-500` displayed after run
- Results tab — SELECT: `<Table>` with scrollable wrapper `max-h-96 overflow-auto`
- Results tab — DML: plain text `{N}개 행이 영향받았습니다. (실행시간: {ms}ms)` at `text-sm`
- Results pagination: client-side, show first 500 rows, display `"결과 {total}개 중 500개 표시"` when truncated
- RBAC enforcement: viewer sees editor but Execute button is hidden; a `<p className="text-xs text-amber-600">` reads `"뷰어 권한으로는 SELECT만 실행할 수 있습니다."` — server also enforces

### /settings (Cloud Storage tab addition)

- Existing `NotificationSettingsForm` becomes one tab; add `<Tabs>` wrapper with tabs: `알림 설정` | `클라우드 스토리지`
- Cloud Storage tab layout matches notification settings form: `max-w-2xl mx-auto`, `space-y-4` fields
- Fields: 엔드포인트 URL, 리전, 버킷명, Access Key ID, Secret Access Key (password input)
- Save button: `<Button>` full-width at bottom of form, label `"저장"`
- When global S3 not configured and schedule modal is open: `<p className="text-xs text-neutral-400">클라우드 업로드를 사용하려면 먼저 설정에서 S3 정보를 입력하세요.</p>` directly below the disabled toggle

## Interaction Contracts

### Dashboard

- Metric cards: non-interactive (display only). No hover state.
- Connection grid rows: clicking a row navigates to `/connections` with that connection highlighted — use `router.push("/connections?highlight={id}")`.
- History list rows: clicking a row opens backup detail (same pattern as `/history` page).
- Auto-refresh: dashboard data does NOT auto-refresh. User refreshes page manually. (Server component — no polling complexity in Phase 4.)

### Query Executor

- Keyboard shortcut: `Ctrl+Enter` (Windows) / `Cmd+Enter` (Mac) triggers SQL execution from within Monaco Editor.
- Execution lock: while query is running, Execute button shows `<Loader2 className="animate-spin" />` and is `disabled`. Prevents double-submit.
- Connection change: switching the connection dropdown clears the results panel but preserves editor content.
- Saving a query: opens a `<Dialog>` with a single text `<Input>` for the query name. Pressing Enter or clicking 저장 confirms.
- Loading a saved query: selecting from the saved queries dropdown populates the editor content. Prompts `"현재 편집 중인 내용이 교체됩니다."` only if editor is non-empty and unsaved.
- Saved query delete: `<AlertDialog>` confirmation required before deletion. Uses `alert-dialog.tsx`.

### Cloud Storage Toggle (ScheduleModal)

- Toggle state: disabled (`disabled` prop) when `CloudStorageSettings` record does not exist in DB.
- Disabled toggle shows hint text below it (see copywriting above).
- Enabled toggle: optimistic update (same pattern as schedule toggle in ConnectionCard).

### Status Badges

Reuse existing badge pattern from Phase 2/3:

| Status | Classes |
|--------|---------|
| 성공 | `text-green-700 bg-green-100 font-medium text-xs px-2 py-1 rounded` |
