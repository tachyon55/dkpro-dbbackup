1. Create `src/lib/validations/user.ts` with Zod schemas:
   ```typescript
   import { z } from "zod"

   export const createUserSchema = z.object({
     email: z.string().email("유효한 이메일을 입력해주세요"),
     name: z.string().min(1, "이름을 입력해주세요").max(50),
     password: z.string().min(8, "비밀번호는 최소 8자입니다").regex(/(?=.*[a-zA-Z])(?=.*\d)/, "영문과 숫자를 포함해야 합니다"),
     role: z.enum(["admin", "operator", "viewer"]),
   })

   export const updateUserSchema = z.object({
     name: z.string().min(1).max(50).optional(),
     role: z.enum(["admin", "operator", "viewer"]).optional(),
     isActive: z.boolean().optional(),
   })
   ```

2. Create `src/app/api/users/route.ts`:
   - **GET**: List all users. Requires admin role (check with `auth()`, return 403 if not admin). Select: id, email, name, role, isActive, lastLoginAt, createdAt. Order by createdAt desc. Do NOT return password field.
   - **POST**: Create user. Requires admin role.
     a. Parse body with `createUserSchema.safeParse()`
     b. Check if email already exists (return 409 "이미 등록된 이메일입니다")
     c. Hash password with bcrypt.hashSync(password, 10)
     d. Create user with `mustChangePassword: true` (per D-02)
     e. Return 201 with created user (exclude password)

3. Create `src/app/api/users/[id]/route.ts`:
   - **GET**: Get single user by id. Admin only. Return user without password.
   - **PUT**: Update user. Admin only.
     a. Parse body with `updateUserSchema.safeParse()`
     b. D-06 guard: if changing role FROM admin or setting isActive=false, count remaining active admins. If count <= 1, return 400 "마지막 관리자 계정은 변경할 수 없습니다"
     c. D-05: if setting isActive=false, just update the field (existing sessions expire naturally per D-05)
     d. Update user, return updated user without password
   - **DELETE**: Delete user. Admin only.
     a. Cannot delete self (return 400 "자기 자신은 삭제할 수 없습니다")
     b. D-06 guard: if target user is admin, count remaining active admins. If count <= 1, return 400 "마지막 관리자 계정은 삭제할 수 없습니다"
     c. Delete user, return 200

4. Create `src/app/api/users/[id]/reset-password/route.ts`:
   - **POST**: Reset password to temporary value. Admin only (per D-08).
     a. Generate a temporary password (e.g., "Temp" + random 6-digit number + "!")
     b. Hash with bcrypt, update user with new password hash and `mustChangePassword: true`
     c. Return 200 with the temporary password in response (admin tells user verbally/securely)

5. All routes MUST re-verify role with `auth()` at the route level (NOT just middleware -- per CVE-2025-29927 anti-pattern from research).
