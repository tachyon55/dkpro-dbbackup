## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, color swatch gaps |
| sm | 8px | Compact element spacing, input icon padding |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding, modal padding |
| xl | 32px | Layout gaps, page content top padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level vertical rhythm |

Exceptions:
- Touch targets (buttons, clickable cards): minimum 44px height for accessibility
- Sidebar width collapsed: 64px; expanded: 240px
- Connection card: fixed height 140px to ensure grid uniformity across 5–12 cards

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Form labels, table cells, descriptions |
| Label | 12px | 600 (semibold) | 1.4 | Badge text, metadata, timestamps, helper text |
| Heading | 18px | 600 (semibold) | 1.3 | Page titles, modal titles, section headings |

**Declared weights:** 400 (regular) and 600 (semibold) only. Two weights total.

**Note:** Display size (24px) is not declared for Phase 1 — reserved for Phase 4 dashboard.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` — white | Page background, modal backgrounds, form surfaces |
| Secondary (30%) | `hsl(240 4.8% 95.9%)` — slate-100 | Sidebar background, card backgrounds, table row stripes |
| Accent (10%) | `hsl(221.2 83.2% 53.3%)` — blue-600 | Primary action buttons, active sidebar item, focus rings, connection test "연결 성공" indicator |
| Destructive | `hsl(0 72.2% 50.6%)` — red-600 | Delete button, account lockout badge, "연결 실패" test indicator, brute-force lockout warning |

Accent reserved for:
1. Primary CTA buttons ("저장", "연결 추가", "사용자 추가")
2. Active/selected sidebar navigation item background
3. Focus ring on all interactive elements (2px outline)
4. Connection test success state icon and response time text
5. "연결됨" status indicator dot on connection cards

**Connection card color accent (CONN-08):** The per-connection user-assigned color is rendered as a 4px left border strip on each card and as a filled 16px circle swatch in the color picker. These are user-defined colors from a preset palette (see Component Inventory below) and are independent of the system accent color.

**Light mode only** — dark mode tokens are not defined in Phase 1. (Source: D-25)

## Interaction Patterns

### Login Page (`/login`)

- Centered card layout, 400px wide, vertically centered in viewport.
- Email input + password input + "로그인" button.
- Error message renders below the button in red text (14px, destructive color).
- Account lockout shows lockout duration remaining ("14분 후 다시 시도하세요").
- No "회원가입" link — admin creates accounts only (per REQUIREMENTS.md Out of Scope).

### Forced Password Change (`/change-password`)

- Same centered card layout as login, 400px wide.
- Fields: 새 비밀번호, 비밀번호 확인.
- Password policy hint shown below input as label/600 gray text (always visible, not on error).
- Submit button: "비밀번호 변경". Disables while submitting.
- On success: redirect to `/connections`.

### Sidebar (D-23, D-24)

- Default state: expanded (240px).
- Toggle button: chevron icon at bottom-left of sidebar, click collapses to icon-only (64px).
- Collapsed state: shows only icons with tooltip on hover.
- Role-gated items (사용자 관리, 감사 로그) render only when `session.user.role === 'admin'`. They do not render at all for non-admins (not disabled, fully hidden).
- Current page: blue-600 background, white text, semibold.
- Inactive page: transparent background, slate-700 text, regular weight, hover: slate-100 background.

### Connection List (`/connections`)

- Page heading: "연결 관리" (18px/600), right-aligned "연결 추가" button.
- Grid: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4`, gap-16px.
- Empty state: centered illustration-free empty state with heading + body + "연결 추가" button.
- Card click: opens ConnectionDetail Sheet from the right (no page navigation).
- Card overflow menu (3-dot icon, top-right of card): shows tooltip "연결 관리" on hover; clicking reveals "수정" and "삭제" items.
- "수정" opens ConnectionModal pre-filled. "삭제" opens shadcn AlertDialog for confirmation.

### Connection Modal (D-10, D-11, D-12)

- shadcn Dialog, max-width 560px, scrollable if content overflows.
- Section 1 — 기본 정보: 연결 이름 (text), DB 타입 (select), 색상 (ColorPicker).
- Section 2 — 접속 정보: fields rendered dynamically per DB type:
  - MySQL/MariaDB/PostgreSQL/SQL Server: 호스트, 포트, 데이터베이스명, 사용자명, 비밀번호
  - SQLite: 파일 경로 (replaces all network fields)
  - Oracle: 호스트, 포트, SID 또는 서비스명 (radio toggle), 사용자명, 비밀번호
- Section 3 — 연결 테스트: "연결 테스트" button. On click: button shows spinner, disabled. Result renders inline below button: success (green check + "연결 성공 · {N}ms") or failure (red X + error message). Test result clears when any field changes.
- Footer: "취소" (ghost variant) + "저장" (default variant, blue). "저장" disabled until required fields filled. Shows spinner during submit.

### User Management (`/users`)

- Page heading: "사용자 관리" (18px/600), right-aligned "사용자 추가" button.
- Full-width table below heading.
- 사용자 추가: opens inline shadcn Dialog (400px) with fields: 이름, 이메일, 역할 (select), 임시 비밀번호 (auto-generated, shown once with copy button).
- Row actions (dropdown): "수정", "비밀번호 재설정", "비활성화"/"활성화", "삭제".
- Last admin protection (D-06): "삭제" and "비활성화" are visually disabled (grayed, tooltip "마지막 admin 계정은 변경할 수 없습니다") when the target is the only active admin.

### Audit Log (`/audit-logs`)

- Page heading: "감사 로그" (18px/600).
- Filter bar (horizontal row above table): 사용자 (select or text search), 이벤트 타입 (select), 시작일~종료일 (date range picker), "초기화" link.
- Table columns: 날짜/시간, 사용자, 이벤트, 대상, IP 주소.
- Pagination: 25 rows/page, "이전" / "다음" buttons, page indicator.
- No delete/export action — read-only view.

### Toast Notifications

- Position: top-right, stacked, max 3 visible.
- Auto-dismiss: 4 seconds.
- Types used: success (green left border), error (red left border), info (blue left border).
- Use shadcn `toast` / sonner — whichever ships with `npx shadcn@latest add toast`.

### Loading States

- Page-level data fetch: shadcn `Skeleton` matching the shape of the loaded content (card skeletons for connection grid, row skeletons for tables).
- Button actions (save, test, login): button shows `Loader2` lucide icon spinning, button disabled during action.
- Sidebar navigation: instant (no loading state needed for route transitions via Next.js App Router).
