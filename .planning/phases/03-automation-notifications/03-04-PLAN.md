# Plan 03-04: Wire Custom Backup Path from Scheduler to Backup Engine

**Goal:** SCHD-03 gap closure — a custom `backupPath` configured on a Schedule must be applied to the actual backup file write, not silently discarded.

## Context

Verification found that `schedule.backupPath` is stored correctly in the database and accepted by the API and UI (ScheduleModal has the path input field), but the value is never applied at execution time.

In `scheduler.ts`, `runScheduledBackup` at line 84 computes `backupDir` from `schedule.backupPath`, then line 85 immediately discards it with `void backupDir` and a comment acknowledging this is deferred. The `runBackup` function in `backup-engine.ts` has signature `runBackup(historyId, send)` — no third parameter — so it always calls `getBackupDir(connectionId)` at line 95 to derive the output directory.

The fix is a two-file change: add an optional `backupDir` parameter to `runBackup`, use it when truthy, and pass the computed `backupDir` from the scheduler to close the wiring gap.
