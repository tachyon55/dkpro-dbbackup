  <files>netlify.toml, vercel.json</files>
  <action>
1. `netlify.toml` 파일을 삭제한다 (git rm 또는 파일 삭제).

2. 프로젝트 루트에 `vercel.json`을 새로 생성한다:

```json
{
  "buildCommand": "npx prisma generate && next build",
  "framework": "nextjs",
  "comments": {
    "socket_io_limitation": "[중요] Socket.io 제한 사항: Vercel은 서버리스 함수 기반으로 동작하므로 영구적인 WebSocket 연결을 지원하지 않습니다. Socket.io를 사용하는 실시간 백업 진행 상황 기능은 Vercel에서 동작하지 않습니다. 실시간 기능이 필요한 경우 Railway, Render, Fly.io 등 장기 실행 서버 환경을 사용하세요."
  }
}
```

참고: vercel.json은 JSON 형식이므로 주석 필드는 실제로는 사용되지 않는 더미 키다. 대신 파일 상단에 실제 주석은 넣을 수 없으므로, Socket.io 제한 사항과 환경변수 안내는 별도 섹션(env 문서화 주석 키)으로 포함한다.

실제 vercel.json 내용:

```json
{
  "buildCommand": "npx prisma generate && next build",
  "framework": "nextjs",
  "env": {
    "DATABASE_URL": "PostgreSQL 연결 문자열 — Vercel 대시보드 > Project > Settings > Environment Variables 에서 설정",
    "NEXTAUTH_URL": "배포된 사이트 URL (예: https://your-site.vercel.app)",
    "NEXTAUTH_SECRET": "NextAuth 서명 키 (openssl rand -base64 32 로 생성)",
    "ENCRYPTION_KEY": "AES-256 암호화 키 32바이트 hex (openssl rand -hex 32 로 생성)",
    "AWS_ACCESS_KEY_ID": "S3 클라우드 스토리지 사용 시",
    "AWS_SECRET_ACCESS_KEY": "S3 클라우드 스토리지 사용 시",
    "AWS_REGION": "S3 리전 (예: ap-northeast-2)"
  }
}
```

단, env 필드에 실제 값이 아닌 설명 문자열을 넣으면 Vercel이 해당 값을 그대로 사용하려 할 수 있어 문제가 생긴다. 따라서 env 필드는 사용하지 않고, 환경변수 안내는 `$schema` 주석 대신 별도 필드로 처리한다.

**최종 vercel.json 내용 (실제로 작성할 파일):**

```json
{
  "buildCommand": "npx prisma generate && next build",
  "framework": "nextjs",
  "_comments": {
    "socket_io": "[중요] Socket.io 제한: Vercel 서버리스 환경에서는 영구 WebSocket 연결이 불가합니다. 실시간 백업 진행 상황 기능은 Vercel에서 동작하지 않습니다. 실시간 기능이 필요하면 Railway, Render, Fly.io 등 장기 실행 서버 환경을 사용하세요.",
    "env_vars": "아래 환경변수를 Vercel 대시보드 > Project > Settings > Environment Variables 에서 설정하세요: DATABASE_URL, NEXTAUTH_URL, NEXTAUTH_SECRET, ENCRYPTION_KEY, AWS_ACCESS_KEY_ID (선택), AWS_SECRET_ACCESS_KEY (선택), AWS_REGION (선택)"
  }
}
```

`_comments`는 Vercel이 무시하는 비표준 키이므로 실제 배포에 영향을 주지 않는다.
